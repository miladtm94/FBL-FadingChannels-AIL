# Guide to Missing `.mat` Files

This document explains the two pre-computed result files that are **not included** in this repository and provides detailed instructions on how to regenerate them.

---

## Why are they missing?

The `.mat` files are generated by running computationally intensive Monte-Carlo simulations (up to 10,000 channel realisations × 30 SNR points × 3 blocklengths). Their sizes range from a few MB to tens of MB, which makes them unsuitable for version control.

They are listed in `.gitignore` under `data/processed/*.mat`.

---

## File 1: `data/processed/FinalResults.mat`

### Purpose
Stores all AIL numerical results for Rayleigh and Rician fading across the full SNR sweep.  
Used by `simulations/run_all.m` (plotting section) to reproduce **Figs. 3–4**.

### Contents

| Variable | Size | Description |
|---|---|---|
| `deltaBar_Rayl_MRT` | 3 × 30 | Exact AIL, Rayleigh, MRT beamforming |
| `deltaBar_Rayl_MRT_SIM` | 3 × 30 | Monte-Carlo AIL, Rayleigh, MRT |
| `deltaBar_Rayl_MRT_Approx` | 3 × 30 | Approximate AIL (Prop. 1), Rayleigh MRT |
| `deltaBar_Rayl_MRT_Approx_HighSNR` | 3 × 30 | High-SNR approx, Rayleigh MRT |
| `deltaBar_Rayl_ANI` | 3 × 30 | Exact AIL, Rayleigh, AN beamforming |
| `deltaBar_Rayl_ANI_SIM` | 3 × 30 | Monte-Carlo AIL, Rayleigh, AN |
| `deltaBar_Rayl_ANI_Approx` | 3 × 30 | Approximate AIL (Prop. 1), Rayleigh AN |
| `deltaBar_Rayl_ANI_Approx_HighSNR` | 3 × 30 | High-SNR approx, Rayleigh AN |
| `deltaBar_Rice_MRT` | 3 × 30 | Exact AIL, Rician, MRT beamforming |
| `deltaBar_Rice_MRT_SIM` | 3 × 30 | Monte-Carlo AIL, Rician, MRT |
| `deltaBar_Rice_MRT_APPROX` | 3 × 30 | Approximate AIL, Rician MRT |
| `deltaBar_Rice_MRT_APPROX_HighSNR` | 3 × 30 | High-SNR approx, Rician MRT |
| `deltaBar_Rice_ANI` | 3 × 30 | Exact AIL, Rician, AN beamforming |
| `deltaBar_Rice_ANI_SIM` | 3 × 30 | Monte-Carlo AIL, Rician, AN |
| `deltaBar_Rice_ANI_Approx` | 3 × 30 | Approximate AIL, Rician AN |
| `deltaBar_Rice_ANI_Approx_HighSNR` | 3 × 30 | High-SNR approx, Rician AN |

**Array dimensions:** rows index `NVec = [500, 5000, 50000]`; columns index `SNRVec_dB = linspace(-20, 10, 30)`.

### How to regenerate

```matlab
% In MATLAB, from repo root:
addpath(genpath('.'));
SNRVec_dB = linspace(-20, 10, 30);
NVec = [5e2, 5e3, 5e4];
filename = 'data/processed/FinalResults.mat';

cd simulations
sim_rayleigh_fading   % ~1–3 hours
sim_rician_fading     % ~2–5 hours (distribution fitting adds overhead)
```

Or simply:
```matlab
run_all   % runs both sequentially
```

## File 2: `data/processed/Results_optimalAST.mat`

### Purpose
Stores the 2-D AST surface over the (α, N) grid for Rayleigh fading.  
Used by `simulations/fig_ast_optimization_surface.m` to reproduce **Figs. 11–12**.

### Contents

| Variable | Size | Description |
|---|---|---|
| `AST_Rayl` | 20 × 100 | AST values; rows = `alphaVec`, cols = `NVec_grid` |
| `alphaVec` | 1 × 20 | Power allocation factor grid `linspace(0.01, 1, 20)` |
| `NVec_grid` | 1 × 100 | Blocklength grid `round(linspace(1, 1000, 100))` |

### How to regenerate

```matlab
addpath(genpath('.'));
cd simulations
fig_ast_optimization_surface   % ~30–60 minutes
```

## Estimated Runtimes

| Script | Approximate Runtime | Hardware assumption |
|---|---|---|
| `sim_rayleigh_fading` | 1–3 hours | Single CPU core, MATLAB R2021a |
| `sim_rician_fading` | 2–5 hours | Single CPU core (dist. fitting adds cost) |
| `fig_ast_optimization_surface` | 30–60 min | Single CPU core |
| `run_all` (both) | 3–8 hours total | Single CPU core |

> 💡 **Tip:** To reduce runtime during development, decrease `simNum` in `config/sys_params.m` (e.g., `simNum = 500`) and reduce the grid resolution in the sweep vectors. Results will be less smooth but the pipeline logic can still be validated.
